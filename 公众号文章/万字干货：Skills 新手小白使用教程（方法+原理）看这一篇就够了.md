### Claude Code 是什么？

Claude Code 是 Anthropic 公司推出的一款官方工具。Anthropic 就是 Claude 大模型的开发商，你可能在网页上用过 claude.ai，那就是他们家的产品

简单来说，Claude Code 是一个运行在"命令行"里的 AI 助手

看到"命令行"这三个字，你可能已经开始紧张了——这是不是程序员才会用的东西？别担心，我来解释一下

**命令行是什么？**

你平时用电脑，大部分操作都是"点点点"——点开文件夹、点开软件、点击按钮。这叫做"图形界面"

命令行则是另一种操作电脑的方式：你输入一行文字指令，电脑就执行相应的操作

比如你想打开一个文件夹，在图形界面里你是双击文件夹图标；在命令行里你是输入 `cd 文件夹名称` ，然后按回车。结果是一样的，只是操作方式不同

命令行听起来很"极客"，但其实非常简单——本质上就是打字。后面的教程里，我会给你准备好所有需要输入的命令，你复制粘贴就行

**为什么 Claude Code 要做成命令行工具？**

你可能会问：既然网页版的 Claude 也能用，为什么还要搞一个命令行版本？这是因为命令行版本有几个网页版做不到的优势：

1. **可以直接操作你电脑上的文件**  
	网页版的 Claude 只能处理你复制粘贴给它的文字。但 Claude Code 运行在你的电脑上，它可以直接读取和修改你的本地文件。比如你有一个文件夹里放着 100 篇文章，你想让 AI 帮你全部整理一遍——用网页版你得复制粘贴 100 次，用 Claude Code 一句话就能搞定
2. **可以自动化执行复杂任务**  
	Claude Code 可以把多个步骤串联起来自动执行。比如"先读取这个文件，然后分析内容，最后生成一份报告并保存"，这种连续操作网页版做不了，Claude Code 可以
3. **可以安装 Skill，获得专业能力**  
	这是最关键的一点。网页版的 Claude 就是一个"裸奔"的通用助手，但 Claude Code 可以通过安装 Skill 来获得各种专业能力——这正是本文要教你的

### Skill 是什么？

在开篇部分，我用了一个类比：AI 是手机，Skill 是 App。现在让我更准确地解释一下。

**Skill 本质上是一套"预设的指令和知识"。**

你可以把它想象成一份详细的"工作手册"。当你安装一个 Skill 之后，AI 就相当于读过了这本手册，知道在特定场景下应该怎么做。

举个例子，一个"公众号写作助手"的 Skill 里面可能包含：

- 公众号文章的标准结构模板
- 10 种经过验证的爆款开头写法
- 标题优化的技巧清单
- 不同类型文章（干货、故事、观点）的写作套路
- 排版规范和注意事项

当你让装了这个 Skill 的 AI 写公众号时，它不是从零开始瞎写，而是基于这些预设的知识来输出。所以结果会更专业、更稳定

**Skill 的本质价值是：把专家经验打包，让 AI 可以复用**

### 使用 Skill 的两个前提

了解了 Claude Code 和 Skill 是什么，你现在应该明白：要使用 Skill，你需要先有一个能运行 Skill 的环境。具体来说，有两个前提条件：

**前提一：安装 Claude Code**  
Skill 是运行在 Claude Code 里的（就像 App 运行在手机里）。所以你首先需要把 Claude Code 安装到你的电脑上。

> **补充说明** ：除了 Claude Code，市面上还有一些类似的开源工具，比如 OpenCode。它们的功能大同小异，也能运行 Skill。本文以 Claude Code 为主来讲解，因为它是官方工具，稳定性和兼容性最好。

**前提二：配置大语言模型**  
这一点很多新手会忽略。Claude Code 本身只是一个"壳"，它需要连接一个大语言模型才能工作。你可以把 Claude Code 想象成一个聪明的"身体"，大语言模型则是它的"大脑"。没有大脑，身体再好也没用。

你可以选择连接不同的"大脑"：

- Anthropic 官方的 Claude 模型（国内也有一些中转站提供 Claude 模型）
- 国内的 DeepSeek、Kimi 等模型
- 通过 OpenRouter 中转使用各种模型

具体怎么配置，后面会详细讲。现在，让我们开始动手

## 四、安装 Claude Code

**终端是什么？**  
终端就是一个可以输入文字命令的窗口。你在里面输入指令，电脑就会执行相应的操作。它的界面通常是黑色或白色背景，上面显示一些文字，看起来有点"黑客风"

不同操作系统里，这个软件的名字略有不同：

- Mac 上叫 **“终端”（Terminal）**
- Windows 上叫 **“命令提示符”** 或 **“PowerShell”**

### 4.2 安装 Claude Code

Node.js 安装好之后，安装 Claude Code 就非常简单了——只需要一条命令

1. 打开终端
2. 输入以下命令，按回车：
	```
	npm install -g @anthropic-ai/claude-code
	```
3. 等待安装完成。过程中你会看到一些文字滚动，这是正常的
4. 当命令执行完毕，光标回到 `%` 或 `$` 后面，就说明安装完成了

**验证安装是否成功**  
输入以下命令：

```
claude --version
```

如果看到版本号输出，说明 Claude Code 安装成功

## 五、配置大语言模型

Claude Code 已经装好了，但现在它还不能工作——因为它还没有"大脑"。这一章，我们来给它接上大脑

### 5.1 为什么需要配置模型？

前面说过，Claude Code 本身只是一个"壳"，真正的智能来自于它背后连接的大语言模型

你可以把这个关系想象成：

- **Claude Code** \= 一台没装系统的电脑
- **大语言模型** \= 操作系统

电脑硬件再好，没有操作系统也开不了机。同样，Claude Code 功能再强，没有配置模型也无法回答你的问题

**配置模型的本质是什么？**  
就是告诉 Claude Code：你要用哪个 AI 服务，以及如何连接它。这需要两个信息：

1. **API 地址** ：AI 服务的"门牌号"
2. **API Key** ：你的"通行证"，证明你有权限使用这个服务

不同的 AI 服务商，API 地址和获取 Key 的方式不一样。下面我会介绍几种主流方案

## 六、理解 Skill 的本质

在第二章，我简单介绍过 Skill 是"给 AI 装的外挂"。现在你已经把 Claude Code 安装好、模型也配置好了，让我们更深入地理解一下 Skill 到底是什么。

### 6.1 Skill 的本质：一份专业的工作手册

想象一下，你新招了一个聪明但没经验的实习生。他很聪明，学什么都快，但刚来公司什么都不懂——不知道公司的工作流程，不知道行业的专业术语，不知道这个岗位的最佳实践

你会怎么带他？

![[公众号文章/assets/万字干货：Skills 新手小白使用教程（方法+原理）看这一篇就够了/5bcbd6d1a41ec707acc9923c5bea538e_MD5.unknown]]

最有效的方式是给他一份详细的工作手册：

- 这个岗位的职责是什么
- 常见任务应该怎么处理
- 遇到问题应该怎么思考
- 有哪些注意事项和禁忌

有了这份手册，他就能快速上手，输出的质量也会更稳定。

**Skill 就是 AI 的"工作手册"。**

从技术角度来说，一个 Skill 通常包含以下内容：

1. **角色定义** ：告诉 AI 它现在扮演什么角色（比如"你是一个资深的公众号运营"）
2. **知识库** ：这个领域的专业知识、最佳实践、常见套路（比如"公众号开头的 10 种写法"）
3. **工作流程** ：完成任务的标准步骤（比如"先分析需求→再列大纲→然后分段写作→最后润色"）
4. **输出规范** ：结果应该长什么样（比如"标题控制在 20 字以内，正文分 3-5 个小节"）
5. **示例参考** ：优秀案例供 AI 学习模仿

当你让装了 Skill 的 AI 做事时，它不是"裸奔"状态下随便发挥，而是基于这些预设的知识来输出。所以结果会更专业、更稳定、更符合预期。

### 6.2 Skill 能做什么？

理论上，Skill 可以覆盖任何知识工作场景。只要能把一个任务的"专业知识"和"工作流程"总结出来，就可以做成 Skill。

### 6.3 Skill 和 Prompt 有什么区别？

![[公众号文章/assets/万字干货：Skills 新手小白使用教程（方法+原理）看这一篇就够了/5bcbd6d1a41ec707acc9923c5bea538e_MD5.unknown]]

看到这里，你可能会有一个疑问：

> “你说的这些，我用 Prompt 不也能做到吗？我把要求写详细一点，把最佳实践告诉 AI，不就行了？搞一个 Skill 出来，不是多此一举吗？”

这个问题问得很好。让我来解释一下 Skill 和 Prompt 的本质区别。  
**先说结论：Skill 是"可复用的、系统化的 Prompt 工程"。**

单次 Prompt 确实可以让 AI 做很多事。但当你需要反复做同一类任务时，Skill 的优势就体现出来了。

**区别一：一次性 vs 可复用**

- **用 Prompt 的方式** ：每次写文章，你都要输入一大段要求（“帮我写一篇公众号文章，主题是 XXX。要求：开头要……”）。下次再写，又要重新输入。
- **用 Skill 的方式** ：所有要求都写在 Skill 里了。你只需要说"帮我写一篇关于时间管理的公众号文章"，AI 自动就知道该怎么写。一个 Prompt 你可能写一次就忘了，但一个 Skill 可以用一辈子。

**区别二：零散 vs 系统化**

- 临时写 Prompt，你能想到什么就写什么，很容易遗漏。
- 而一个好的 Skill，是经过系统梳理的。它包含所有环节、最佳实践、避坑指南和优秀案例。这种系统性，是临时写 Prompt 很难做到的。

**区别三：个人经验 vs 专家经验**

- 你自己写 Prompt，用的是你自己的经验。但你可能并不是每个领域的专家。
- 但如果有一个小红书运营专家，把他多年的经验整理成了一个 Skill——你只要用这个 Skill，就相当于拥有了他的经验。Skill 本质上是"专家经验的封装"。

**区别四：不稳定 vs 稳定**

- 同一个 Prompt，AI 的回答可能完全不一样（随机性）。
- Skill 因为包含了详细的规范和示例，可以大幅降低这种不稳定性，输出更稳定。

**区别五：上下文消耗**

- 每次对话，AI 能处理的内容长度是有限的。如果你每次都在 Prompt 里写一大段要求，就会占用很多上下文空间。
- Skill 的加载方式更高效——它只在需要时才加载相关内容，而且是经过优化的精简版本。

**什么时候用 Prompt，什么时候用 Skill？**

```
┌──────────────────────────────────┬──────────────────┐ │               场景               │     推荐方式     │ ├──────────────────────────────────┼──────────────────┤ │ 一次性的、简单的任务             │ 直接写 Prompt    │ ├──────────────────────────────────┼──────────────────┤ │ 探索性的、不确定怎么做的任务     │ 先用 Prompt 试试 │ ├──────────────────────────────────┼──────────────────┤ │ 反复做的、有固定套路的任务       │ 用 Skill         │ ├──────────────────────────────────┼──────────────────┤ │ 需要专业知识的任务               │ 用专家做的 Skill │ ├──────────────────────────────────┼──────────────────┤ │ 对输出质量要求高、需要稳定的任务 │ 用 Skill         │ └──────────────────────────────────┴──────────────────┘
```

**一个类比**  
如果把 AI 比作一个厨师：

- **Prompt** 就像你口头告诉他"今天做个红烧肉，要软烂一点，不要太甜"。
- **Skill** 就像给他一本《中餐烹饪大全》，里面有红烧肉的标准食谱、火候控制要点、摆盘技巧、常见失误避坑指南。

口头说当然也能做出菜来，但有了菜谱，出品会更稳定、更专业。所以，Skill 不是"多此一举"，而是"一劳永逸"。

### 6.4 Skill 存放在哪里？如何被加载？

了解 Skill 的存放机制，有助于你后面安装和管理技能。

**Skill 的存放位置**  
Claude Code 的 Skill 统一存放在一个特定的文件夹里：

- **Mac** ： `/Users/你的用户名/.claude/skills/`
- **Windows** ： `C:\Users\你的用户名\.claude\skills\`

注意这个路径中的 `.claude` 是一个隐藏文件夹（以点开头的文件夹在系统中默认不显示）。

**一个 Skill 长什么样？**  
每个 Skill 是一个独立的文件夹，里面包含一些文件。最核心的是一个叫 `SKILL.md` 的文件，它就是前面说的"工作手册"，用一种叫 Markdown 的格式写成。

一个典型的 Skill 文件夹结构：

![[公众号文章/assets/万字干货：Skills 新手小白使用教程（方法+原理）看这一篇就够了/5bcbd6d1a41ec707acc9923c5bea538e_MD5.unknown]]

**Claude Code 如何识别和加载 Skill？**  
当你在 Claude Code 中执行某个任务时，它会：

1. 扫描 skills 文件夹，看看有哪些可用的技能
2. 根据你的输入，判断是否需要调用某个技能
3. 如果需要，就读取对应的 `SKILL.md` ，把里面的知识"加载"到上下文中
4. 然后基于这些知识来完成你的任务

你也可以主动告诉 Claude Code 使用某个技能，比如输入"用社交帖子撰写助手帮我写一条帖子"。

**两种安装 Skill 的方式**

- **方式一：命令安装**  
	你可以让 Claude Code 帮你安装。比如：“帮我安装下载目录里的 social-post-writer 技能”。Claude Code 会自动找到文件，并复制到正确的位置。
- **方式二：手动安装**  
	你也可以直接把技能文件夹复制到 skills 目录下。效果是一样的。
1. 下载并解压技能文件
	2. 打开 skills 文件夹（Mac: `~/.claude/skills/` ，Windows: `%USERPROFILE%\.claude\skills\` ）
	3. 把技能文件夹拖进去
	4. 完成
