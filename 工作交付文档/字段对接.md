## 问题背景：
单从电子病历来说，和我们病案机器人的要求相比，还缺少：
1 查房记录(细化可能用到)
2 日常病程记录(文档看起来只给了首次病程记录)
3 检查报告
4  检验记录
5  分娩记录
6  收费明细
7  病理报告
找到对方表字段与我们需要报告类型的对应关系
## 一）查房记录/日常病程记录（通过`rcd_time记录日期时间`区分）

| 信息项      | 说明           | 对应接口字段（coursrinfo）            |
| -------- | ------------ | ----------------------------- |
| 记录日期时间   | 查房的具体时间      | `rcd_time`                    |
| 科室代码     | 查房科室         | `dept_code`                   |
| 科室名称     |              | `dept_name`                   |
| 病区名称     |              | `wardarea_name`               |
| 病床号      |              | `bedno`                       |
| 查房医师     | 上级医师/住院医师    | `ipdr_name` / `prnt_doc_name` |
| 查房医师编号   |              | `ipdr_code` / `prnt_doc_code` |
| **主诉**   | 患者主要症状描述     | `chfcomp`                     |
| **病例特点** | 病史、体征、检查摘要   | `cas_ftur`                    |
| **诊疗计划** | 具体医嘱、检查、治疗安排 | `dise_plan`                   |
| 治则治法     | 中医治疗方案       | `prnp_trt`                    |
**tips：
- 病例特点通常包含：
	- 入院时主要情况
	- 诊疗经过摘要
	- 目前症状、体征变化
	- 已做的检查结果
- 诊疗计划通常包括：
	- 护理级别调整
	- 饮食要求
	- 检查项目安排
	- 药物治疗方案
	- 手术安排
