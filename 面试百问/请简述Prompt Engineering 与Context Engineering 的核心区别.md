# Prompt Engineering 与 Context Engineering 核心区别解析
![[核心区别.png]]

## 1. 核心概念对比

1\. **Prompt Engineering**  
- 标签：How to think  
- 核心功能：挖掘模型内在能力，引导模型思考方式  
- 类比：教导实习生"如何工作"的方法论  
- 关键技术：  
  - Chain-of-Thought (COT)：分步推理引导  
  - Few-shot：示例示范  
- 本质：激活模型参数中已存储的知识和逻辑能力

2\. **Context Engineering**  
- 标签：What to know  
- 核心功能：管理信息输入流  
- 类比：为实习生提供工作所需的"具体资料"  
- 关键技术：  
  - Retrieval-Augmented Generation (RAG)：精准检索增强  
  - 信息窗口管理：有限上下文的高效利用  
- 本质：解决模型无法记忆全部世界知识的问题


## 2. 工程实践痛点与解决方案

![[Prompt Engineering挑战.png]]

### 2.1 Prompt Engineering挑战
1\. **脆弱性问题**  
- 现象：微小语法变化(如句号改逗号)导致输出剧变  
- 解决方案：  
  - 自动化优化：将Prompt作为代码参数自动搜索最优解  
  - 系统化测试：建立Prompt测试框架
2\. **复杂推理问题**  
- 现象：模型容易跳步、产生幻觉(hallucination)  
- 解决方案：  
  - COT + Self-consistency：  
    - 强制分步推理  
    - 生成多条推理路径投票表决  
### 2.2 Context Engineering挑战
1\. **中间信息丢失(Lost in the Middle)**  
- 现象：模型倾向忽略上下文中间部分的关键信息  
- 解决方案：  
  - 重排策略：将关键信息强制置于Prompt首尾  
  - 信息分层：重要性分级嵌入
2\. **噪声污染问题**  
- 现象：检索到无关信息导致模型输出偏差  
- 解决方案：  
  - Self-refund：模型先自检材料相关性  
  - 过滤机制：前置信息质量评估

## 3. 高层架构总结
![[三大支柱理论.png]]
1\. **三大支柱理论**  
- 思维支柱(Prompt)：解决"How"的问题  
  - 教学方法论与推理逻辑  
- 知识支柱(Context)：解决"What"的问题  
  - 精准信息供给与管理  
- 代理支柱(Agent)：未来方向  
  - Prompt与Context的动态融合  
  - 自主任务规划与记忆管理

2\. **协同效应**  
- 优质Prompt需要相关Context支撑  
- 充分Context需要合理Prompt引导  
- 比喻："如同建造房屋，两者是缺一不可的支柱"
